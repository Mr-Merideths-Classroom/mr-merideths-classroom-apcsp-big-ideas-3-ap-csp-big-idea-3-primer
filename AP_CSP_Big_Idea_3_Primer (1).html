3.1

Variables and AssignmentsStoring and Changing Data

📦
What is a Variable?
3.1.A
▼

A variable is a named storage location in memory that holds a value. That value can be a number, text, true/false, or more complex data. The variable's name lets you refer to the stored value throughout your program.

// Assignment: name ← value (AP pseudocode notation) score ← 0 playerName ← "Alex" isActive ← true // Re-assignment changes the value score ← score + 10 // score is now 10
Exam TipWhen a variable is assigned a new value, the old value is lost. The right-hand side is evaluated first, then stored in the variable on the left.

3.2

Data AbstractionManaging Complexity with Lists

🗃️
Lists and Data Abstraction
3.2.A / 3.2.B
▼

Data abstraction means grouping multiple values under one name. A list stores an ordered collection of items. Instead of ten separate variables, you use one list — making the program simpler to write and change.

scores ← [85, 92, 78, 95, 88] // Access by index (AP pseudocode uses 1-based indexing!) OUTPUT(scores[1]) // outputs 85 OUTPUT(scores[3]) // outputs 78 // Useful list operations APPEND(scores, 100) // adds 100 to the end INSERT(scores, 2, 90) // inserts 90 at index 2 REMOVE(scores, 1) // removes item at index 1 LENGTH(scores) // returns number of items
Watch OutAP CSP pseudocode uses 1-based indexing — the first element is at index 1, not 0. This differs from most real programming languages like Python and JavaScript.

Data abstraction manages complexity because you can process any number of items with the same code. A loop over a 5-item list works identically on a 500-item list.

Conditionals & Nested ConditionalsMaking Decisions

🔀
IF / ELSE IF / ELSE Structures
3.6.A / 3.7.A
▼

Selection lets a program take different paths based on conditions. A nested conditional is an IF statement placed inside another IF statement, allowing more complex branching logic.

// Basic conditional IF (score >= 90) { DISPLAY("A") } ELSE IF (score >= 80) { DISPLAY("B") } ELSE { DISPLAY("C or below") } // Nested conditional IF (isLoggedIn) { IF (isAdmin) { DISPLAY("Admin panel") } ELSE { DISPLAY("User panel") } } ELSE { DISPLAY("Please log in") }
Exam TipOnly ONE branch executes in an IF / ELSE IF / ELSE chain — the first condition that evaluates to true. Nested conditionals are only reached if the outer condition is true.

3.8

IterationLoops and Repetition

🔄
REPEAT, REPEAT UNTIL, FOR EACH
3.8.A / 3.8.B
▼

Iteration is the repetition of a block of code. Loops are used to process lists, repeat actions, and avoid writing the same code multiple times.

// REPEAT n TIMES — runs exactly n times REPEAT 5 TIMES { DISPLAY("Hello") } // REPEAT UNTIL — runs until condition becomes true count ← 1 REPEAT UNTIL (count > 5) { DISPLAY(count) count ← count + 1 } // FOR EACH — iterates over every item in a list scores ← [80, 90, 70] FOR EACH item IN scores { DISPLAY(item) }
Infinite LoopsA REPEAT UNTIL loop that never reaches its exit condition runs forever. Always verify that the condition can eventually become true.

3.11

Binary SearchSearching Sorted Data Efficiently

🔍
Linear Search vs. Binary Search
3.11.A
▼

Linear search checks each element one-by-one from the beginning — works on any list. Binary search repeatedly halves the search space — works only on sorted lists but is dramatically faster.

AlgorithmRequires Sorted?Max Steps (n items)Best Case
Linear searchNon steps1 (first item matches)
Binary searchYeslog₂(n) steps1 (middle item matches)
Binary Search ExampleSearching a sorted list of 1,000 items: linear search takes up to 1,000 steps. Binary search takes at most 10 steps (log₂1000 ≈ 10). For 1,000,000 items: binary search takes at most 20 steps.

Binary search works by checking the middle element. If too high, search the left half. If too low, search the right half. Repeat until found or the search space is empty.

binary searchlinear searchsorted listlog n
3.12 – 3.13

Calling & Developing ProceduresFunctions and Abstraction

⚙️
Procedures, Parameters, and Return Values
3.12.A / 3.13.A
▼

A procedure (function) is a named, reusable block of code. Parameters are inputs to the procedure. A return value is the output. Procedures reduce repetition and manage complexity by giving a name to a multi-step process.

// Defining a procedure with a parameter and return value PROCEDURE getGrade(score) { IF (score >= 90) { RETURN "A" } ELSE IF (score >= 80) { RETURN "B" } ELSE { RETURN "C or below" } } // Calling the procedure result ← getGrade(87) // result = "B" DISPLAY(getGrade(95)) // displays "A"
Procedural AbstractionWhen you call a procedure, you don't need to know how it works internally — only what inputs it takes and what it returns. This is the essence of abstraction.
TermDefinition
ParameterVariable in the procedure definition that receives an argument
ArgumentActual value passed to the procedure when calling it
Return valueValue the procedure sends back to the calling code
PROCEDURERETURNparameterargumentabstraction
3.14

LibrariesUsing Pre-Built Code

📚
APIs and Libraries
3.14.A
▼

A library is a collection of pre-written procedures that programmers can use. An API (Application Programming Interface) specifies how to use a library — the procedure names, parameters, and return values — without exposing the internal implementation.

Why Libraries MatterLibraries allow programmers to build on existing work rather than solving every problem from scratch. You can use a random number library without understanding how random numbers are generated internally.

Using a library procedure: you must know the specification (what inputs it takes, what it returns), but not the implementation (how it works internally).

libraryAPIspecificationabstraction
3.15

Random ValuesIntroducing Unpredictability

🎲
RANDOM and Non-Deterministic Behavior
3.15.A
▼

RANDOM(a, b) returns a random integer from a to b inclusive. Programs that use random values are non-deterministic — the same inputs can produce different outputs on different runs.

// Simulate a coin flip flip ← RANDOM(1, 2) IF (flip = 1) { DISPLAY("Heads") } ELSE { DISPLAY("Tails") } // Roll a six-sided die roll ← RANDOM(1, 6)
Exam TipRANDOM(1, 6) can return 1, 2, 3, 4, 5, or 6. Both endpoints are included. This is important for calculating probabilities.
RANDOMnon-deterministicsimulation

3.16.A / 3.16.B

A simulation is a program that models a real-world process. Simulations are used when the real process is too dangerous, expensive, or time-consuming to test directly (e.g., climate modeling, crash testing, disease spread).

FeatureSimulationReal-World Experiment

SafetyNo real riskMay be dangerous

Limitations of SimulationsSimulations are only as accurate as the assumptions built into them. If the model oversimplifies reality, the results can be misleading. Always question what a simulation leaves out.

▼

An algorithm's efficiency describes how its runtime grows as input size increases. The AP exam distinguishes between reasonable (polynomial) and unreasonable (exponential) run times.

Undecidable ProblemsThe Limits of Computing

∞
The Halting Problem & Undecidability
3.18.A
▼

Some problems are undecidable — no algorithm can solve them for all possible inputs, even with unlimited time and memory. This is a fundamental limit of computing, not just a technology limitation.

The Halting ProblemCan we write a program that determines whether any arbitrary program will halt (finish running) or run forever? Alan Turing proved in 1936 that this is impossible — no such program can exist. This is the most famous undecidable problem.
CategoryDefinitionExample
DecidableAn algorithm exists that always gives a correct yes/no answerIs this number prime?
UndecidableNo algorithm can solve it for all inputsWill this program halt?
Key DistinctionAn undecidable problem is different from an unsolved problem. It's not that we haven't found the algorithm yet — it's been proven that no such algorithm can exist.
undecidablehalting problemTuringlimits
